RAG智能客服

发表于2026-05-13|更新于2026-05-13|项目经历
|浏览量:

前言

现在ai发展是在是太快了,但是python的学习又要花费我的一些时间,所以还是就基于我现有的知识框架,
通过学习Java目前比较成熟的Langchain4J进行ai相关知识的学习,基于对Langchain4J的学习,
做了一个RAG智能客服系统

技术框架

这个系统还是基于SpringBoot框架下进行开发的,还有一些基本的Mybatis-Plus,MySQL等,
用到的一些其他新技术就是Langchain4J,MongoDB这两个,
其他的细节技术可以在Langchain4J入门系列里面详细了解

项目描述

  • 采用 LangChain4j + DeepSeek API 构建 AI 对话能力，通过本地 Markdown 知识库实现 RAG 检索增强生成，解决医疗场景下的专业问答准确性问题
  • 使用 MongoDB 存储多会话聊天记忆，结合 memoryId 实现上下文连贯的对话交互，减少重复信息获取
  • 基于 MyBatis-Plus 封装预约挂号模块，支持 AI 自动收集患者信息（姓名、身份证、科室、日期）并完成预约登记
  • 通过 Knife4j 生成 API 文档，提供可视化的接口调试能力，便于前后端联调

总结

现在的AI发展实在是太快了,想当年GPT-3刚出来,就是大轰动,现在的GPT 5,Gemini 3,Claude 4系列等,
简直就是百花齐放,只能说不能停滞的学习老版本知识体系,跟着时代的发展,学习主流技术,才是以后发展的必要条件
