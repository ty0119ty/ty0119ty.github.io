前言

上一篇文章讲到了如何实现AI基础配置以及聊天记忆的持久化,本文章再讲解一些其他的用法

提示词(Prompt)

在大语言模型中,Prompt 的质量会直接影响回答的质量;
问题描述越清晰、约束越明确,模型给出的回答通常也越准确;

System Prompt (@SystemMessage)

仔细阅读上一篇我们知道Assistant就是一个助手,那么我们提前给整个助手加上限定,
比如

  1. 你是一个医生,只能回答医学相关内容
  2. 你是一个记者,回答的新闻时事必须为实际发生的事,不要虚构捏造

等等一系列的限定,那么Assistant之后就只会回复相关内容了,这就是系统提示词的作用

如果系统提示词发生变化,旧的聊天记忆可能与新的系统设定不一致,
因此在实际开发中通常会重新创建新的 ChatMemory

User Prompt(@UserMessage)

有了系统提示词之后,相当于就是给Assistant加了一道身份,但是如果用户需要一些更”优质”的提示,
比如: 我是你的好朋友/我是一名病人,请用温柔的语气和我交流

但是总不可能让用户每次询问都加上这段话吧,那么这就是@UserMessage的作用了,
@UserMessage 用于定义用户消息模板,LangChain4j 会将方法参数填充到模板中,并作为用户消息发送给模型;

提示词变量 (@V)

@V 用于给 Prompt 模板中的变量赋值。
在 @SystemMessage 或 @UserMessage 中可以使用 作为占位符

我们可以在系统提示词@SystemMessage和@UserMessage里面添加{{**}}这样的模板信息,
然后通过@V()进行添加,比如在系统系统提示词里面:

我是,今年岁了,请用重庆话回答我
今天是

然后在chat()方法里面加上参数

工具(Tools)/Function Calling

大模型并不一定擅长在特定领域的方法,所以这部分方法需要我们自己去写,
比如大模型对大量数据运算存在部分误差,那么我们需要自己写计算程序

使用

  1. 在@AiService注解里面加上参数tools,然后流程和Provider差不多,也是去配置好
  2. 需要在Tools类里面的方法加上@Tool的注解,才能让Assistant识别到这个
  3. 命名一定要规范,比如加法运算就用add/sum这样的方法名,如果不好命名,就加上注解 例如:

    @Tool(name = “加法运算”,value = “将a与b两个数字相加”)

  4. 如果想对参数进行描述,就在参数前面加上@P注解
