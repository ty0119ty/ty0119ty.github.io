中科软面经

岗位JD:

  • 从事金融领域的软件产品开发，为行业提供产品解决方案和产品实施服务。
  • 岗位职责：
    • 1.负责公司软件项目以及产品的设计与开发；
    • 2.根据开发进度和任务分配，完成相应的软件的设计、开发、编码、上线工作；
    • 3.对项目中的客户需求或内部需求进行分析、整理、并负责分解到相关的功能模块，为进行模块设计提供相关的依据，并参与
      开发过程中各阶段的评审；
    • 4.配合项目经理完成其他相关任务目标；
  • 任职要求：
    • 1.学历：全日制统招本科及以上学历，专业：计算机软件或相关专业；
    • 2.专业技能：
      • ①掌握WEB应用相关开发技术，掌握JAVA开发技术；
      • ②熟悉至少一种大型数据库开发，如SQLSERVER/Oracle/MySQL等
      • ③熟练掌握使用以下技术：JSP、Servlet、JDBC等企业级应用开发
      • ④精通或熟练使用SSM、SSH、SpringBoot、swagger、shiro、Mybatis等相关框架技术
        S精通或熟练使用Js、Jquery、EasyUI、CSS、DIV、html5、bootstrap，angularjs、vue、ant design等前端技术
    • 3.其他：
      • ①有团队合作经验，良好的沟通、协调能力，强烈的责任感；
      • ②思路清晰，做事有条理，较强的独立处理问题的能力；
      • ③有保险或者金融知识及系统开发集成经验优先，获得专业相关奖励优先；

🧠 优化版

最大的挑战是刚接触真实业务时，发现很多场景和学习阶段不一样。

5️⃣ 500ms → 50ms（你这题是加分项，但没说透）
❌ 问题

没说“为什么慢”

没说“哪些接口”

没说“怎么验证”

🧠 标准加分答案（重点背）

这个数据主要来自我们对老人信息和健康数据的高频查询接口优化前后的对比。

原来接口每次请求都会直接访问数据库，在并发稍高时响应时间在 400～500ms。

后来我引入 Redis 作为缓存层，对 读多写少的核心接口做了缓存设计，
使用 key + 业务维度 作为缓存 Key，并设置合理的过期时间。

在优化前后，我通过 Swagger 和日志打印方式对比接口响应时间，
优化后接口稳定在 50ms 左右，数据库访问压力明显下降。

👉 这已经是真实项目级回答

6️⃣ 权限模型（你方向是对的）
🧠 优化版

系统采用的是 RBAC（基于角色的访问控制）模型，
核心是 用户 – 角色 – 权限 三层结构。

用户不直接绑定权限，而是通过角色获取权限，
比如护士角色只具备老人信息的查看和录入权限，
管理员角色才有系统配置和权限管理能力。

7️⃣ 新增家属端权限（你答得其实不错）
🧠 稍微包装一下

家属端会作为一个独立角色存在，
首先通过绑定关系确定家属和老人之间的关联。

家属角色只允许访问所绑定老人的信息，
比如健康数据、设备状态等，不具备任何数据修改权限，
通过权限隔离避免越权访问。

8️⃣ 角色如何分配权限（你这题是“没懂问题”）
面试官真正想问的是：

权限是怎么落到接口上的？

🧠 正确答案

权限是以 接口或功能维度进行定义的，
角色和权限通过中间表进行关联。

在 Spring Security 中，通过注解或配置方式
将接口和权限标识绑定，请求到来时根据用户角色进行鉴权。

9️⃣ 数据库建表注意点（勉强及格）
🧠 优化版

建表时主要考虑：

字段类型是否合理，避免浪费空间

是否需要冗余字段来减少关联查询

是否需要为高频查询字段建立索引

预留扩展字段，比如状态位、逻辑删除字段

🔟 主键 / 外键
🧠 正确说法

一张表通常只有一个主键，用于唯一标识一条记录；
外键用于维护表之间的关联关系，
但在高并发系统中，更多是通过业务层控制关系，
减少数据库层面的强约束以提升性能。

1️⃣1️⃣ 事务（这是你这场面试最大的技术短板）
🧠 下次必须这样答

MySQL 的事务主要是为了保证 ACID 特性，
在 InnoDB 中通过 redo log 和 undo log 来实现原子性和持久性，
通过 MVCC 和锁机制来保证隔离性。

在实际开发中，我主要关注事务边界的控制，
避免事务过大影响并发性能。

1️⃣3️⃣ 最近学习内容
🧠 优化版

最近在系统学习 消息队列相关内容，
主要是 RabbitMQ 的使用场景、消息可靠性以及异步解耦，
希望在后续实习中能接触到更完整的系统架构。
