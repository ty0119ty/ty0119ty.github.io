前沿

先写Skill这部分的八股是因为其实最近在学习skill,并且面经里面也有这一部分的相关问题,
所以就先写这部分咯

基础

Skills是什么

Skills 简单来说就是一套预定义的指令集,它告诉 AI 在遇到特定类型的任务时应该怎么做、遵循什么规范、输出什么格式

  • Skills 是一个文件夹,包含 SKILL.md 文件（必需）和可选的资源文件
  • SKILL.md 包含 YAML 元数据（name、description）和 Markdown 指令
  • Claude 根据任务需要,动态加载相关 Skill 的内容
对比项 Function Calling MCP Skills
本质 API调用 工具协议 知识包
格式 JSON 复杂协议 Markdown+YAML
Token消耗 低(只传参数) 高(协议开销) 按需加载
适用场景 调用外部API 复杂工具集成 增强特定任务能力
学习成本 中 高 低

完整的skills那样还有scripts,assets,references文件夹

skills一般放在哪里

一般存放在在智能体路径下的skills文件夹下

例如Claude code 就是在 ~/.claude/skills/

为什么不直接将Skills的内容加载进去

  1. 这个涉及到大模型的一个限制就是上下文窗口是有限的,将skill全加载进去会占用大量的上下文空间
  2. 注意力稀释问题,大模型在处理长上下文时,信息越多,对每条指令的关注度越低,只加载当前任务相关的 Skill,模型的注意力更集中,执行质量也更高
  3. 性能和成本,token的数量直接影响了推理速度以及API调用量,所以按需加载对应是skill是更优质的选择

把Prompt直接复制粘贴用不就行了,为什么还要skill呢

    • 复制粘贴 Prompt 最大的问题在于上下文管理成本高,prompt过长可能也需要微调,版本变化没有更新日志
    • Skills 解决的核心问题是自动化触发和结构化管理,当用户提到的时候大模型会自己调用skill
  2. 可复用性,可以团队共享,保证整个团队输出质量相当
  3. 可组合性,复杂任务同时调用多个skill即可,光靠复制粘贴很难做到这点

如何创建一个SKILL

核心就一个,写好SKILL.md文档,然后放在指定目录下即可

写好 SKILL.md 之后,需要给它配一个触发描述,写得越精准,触发的准确率越高

你自己设计过哪些Skill

我设计过一个新项目快速上手的Skill,
这个skill的场景是当用户到手一个新项目之后,让ai直接辅助用户快速了解项目,
包括项目框架,技术栈,还有项目的学习路线,以及后续的项目问答

创建skill需要注意的事项

  1. 触发描述一定要写精准
  2. 指令要具体到可执行,不能太抽象
  3. 要给好的示例
  4. 控制 Skill 的粒度
  5. 注意指令的优先级
  6. 持续迭代

未来skill怎么发展

  1. 标准化,现在每个团队/公司使用的Skill格式可能不同,未来可能会有一种行业标准吧,类似于npm这样的
  2. 利益化,个人开发者将自己编写的Skill发布到商店,让用户付费/免费使用,这样可能会带来一个新的市场吧

你认为未来程序员的核心竞争力是什么

  • 定义问题能力,现在的ai都是帮我们解决问题,但是前提是我们能准确的去定义这个问题,让ai执行更高效
  • 系统设计能力,ai只是执行者,而人才是决策者
  • 终身学习能力,ai时代发展迅速,今天可能是skill,明天可能就过时了,保持学习,保持核心竞争力
