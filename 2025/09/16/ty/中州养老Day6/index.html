Redis的基本数据类型

这下面就是redis的基本数据类型,现在还不知道哪些常用噢

各种数据类型的常用命令

字符串(String)

SET key value (设置指定key的值)

GET key (获取指定key的值)

SETEX key seconds value (设置指定key的值,并将key的过期时间设置为seconds秒)

SETNX key value (只有在key不存在时设置key的值)

注意:

  1. SET key(同一个),那么原来的key会被覆盖
  2. del key,删除操作是通用命令
  3. setnx和set在存储数据的时候,set可以覆盖原有数据,但是setnx不可以

哈希(Hash)

Hash类型,也叫散列,其value是一个无序字典,类似于Java中的HashMap结构
(我连这个HashMap都不了解,也是无敌了),
Redis hash是一个String类型的field和value的映射表,hash特别适合用于存储对象

HSET key field value (将哈希表key中的字段field的值设置为value)

HGET key field (获取key中field的值)

HDEL key field (删除key中field的值)

HKEYS key (获取哈希表中所有字段)

HVALS key (获取哈希表中所有值)

哈希表的数据大概长这样

列表(List)

Redis中的List类型类似于Java中的LinkedList,可以看作是一个双向链表结构,支持正向/反向检索,
特征也与LinkedList类似: 有序,元素可重复,插入删除快,查询速度一般

LPUSH key value1 [value2] (将一个或多个值插入列表头部)

LRANGE key start stop (获取列表指定范围内的元素)

RPOP key (移出并获取列表中最后一个元素)

LLEN key (获取列表长度)

集合(Set)

Redis的Set结构与Java中的HashSet类似,可以看作是一个value为null的HashMap,
因为也是一个哈希表,因此具备HashSet类似的特征:
无序,元素不重复,查找快,支持交并集等功能

SADD key member1 [member2] (向集合中添加一个或多个成员)

SMAMBERS key (返回集合中的所有成员)

SCARD key (获取集合的成员数)

SINTER key1 [key2] (返回给定所有集合的交集)

SUNION key1 [key2] (返回给定所有集合的并集)

SREM key member1 [member2] (移出集合中的一个或者多个成员)

有序集合(SortedSet/zset)

Redis的SortedSet是一个可排序的set集合，与Java中的TreeSet有些类似,
SortedSet中的每一个元素都带有一个score属性，可以基于score属性对元素排序,
SortedSet具备下列特性:可排序、元素不重复、查询速度快

ZADD key score1 member1 [score2 member2] (向有序集合添加一个或多个成员)

ZRANGE key start stop [WITHSCORE] (通过索引区间返回有序集合中指定区间内的成员)

ZINCRBY key increment member (有序集合中对指定成员的分数加上增量increment)

ZREM key member1 [member2] (移除有序集合中的一个或多个成员)

通用命令

不分数据类型,所有都可以使用

KEYS pattern (查找所有符合给定模式(pattern)的 key)

EXISTS key (检查给定 key是否存在)

TYPE key (返回 key所储存的值的类型)

DEL key (该命令用于在 key存在是删除 key)

Spring Cache

一个框架,导入这个依赖,就能实现缓存功能,也要通过maven导入

Spring Cache常见注解

  1. @EnableCaching (放在启动类上面,开启项目缓存功能)
  2. @Cacheable (先在缓存查,再去数据库查,保存到缓存里面)
  3. @CachePut (将返回值添加到缓存里面)
  4. @CacheEvict (将一条/多天数据从缓存里面删除)
  5. @Caching (缓存的结合体,可以组合以上注解,包括:新增删除)

第五点这么用的:

wtm看见视频里面的查询,从数据库里面查996ms,到缓存里面就是16ms了,逆天了,
优化这么大吗,666 27ms–> 9ms,太jb明显了

导入这个@Cache的时候要注意,我导入成了
import springfox.documentation.annotations.Cacheable;
这个是错的,应该是
import springfox.documentation.annotations.Cacheable;
不然就要有错,在key=”#***”那里,说没有key()方法….无语ing

最基本的思路

  • 查询添加到缓存种
  • 增删改的时候删除缓存

因为后面增删改时候还是要查,查的时候就会把新数据插入到缓存中去了
