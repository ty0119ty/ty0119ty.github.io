70.爬楼梯(easy)

动态规划,记忆化搜索,数学

题目

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。
每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

解析

可以这样想,想要爬到第n阶,其实就是从[n-1]阶+1或者[n-2]+2这两种方法; 即
f(n)=f(n-1)+f(n-2)
这不就是斐波那契数列吗?
所以用代码实现

总结

这就是一个简单了斐波那契数列问题,看出来了就会做得很快,不然就有点费脑子了,
洛谷上面也有一个三角形问题,也是斐波那契数列问题
https://www.luogu.com.cn/problem/T676367?contestId=281480

长为b（厘米）的铁丝，4≤b≤1000，要截成n小段（n≥3），
每段的长度不小于1厘米，如果其中任意三小段都不能拼成三角形，则n的最大值为几？

198.打家劫舍(medium)

数组,动态规划

题目

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，
影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，
如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。
给定一个代表每个房屋存放金额的非负整数数组，计算你不触动警报装置的情况下 ，
一夜之内能够偷窃到的最高金额。

解析

有思路,但是写不出来,就是判断现在这家home[i]该不该偷,怎么判断呢,
只需要判断home[i-1]和home[i-2]+home[i]的金额哪个更大即可,
但是我写不出来啊,我的天啊,还是没理细致,慢慢来

理解money[i]=Math.max(money[i-1], money[i-2]+nums[i]);这段代码:

  1. 假设现在有5家人
  2. 当第三家应该偷时,那么money[2]=money[0]+nums[2]
  3. 现在对于第四家来说,只有也是只有两种选择,偷/不偷
  4. 假设偷,那么money[3]=money[1]+nums[3],如果后面都这样,一直这样循环就可以了
  5. 假设不偷,那么money[3]=money[2]
  6. 那么第5家包偷的,毕竟不可能money[4]=money[3]>money[2]+nums[4] 吧
  7. 所以要理解一下,并不会跳过某一个值之类的情况出现

总结

其实总结也就是理解一下那段代码了,而且还有这些类似的动态规划,我发现好像只需要找到规律就行了,
考虑第i种情况是怎么构成的,在根据这个进行写代码,
说着简单,要是真做起来了又只有木起了

279.完全平方数(medium)

广度优先搜索,数学,动态规划

题目

给你一个整数 n ，返回 和为 n 的完全平方数的最少数量 。
完全平方数 是一个整数，其值等于另一个整数的平方；换句话说，其值等于一个整数自乘的积。
例如，1、4、9 和 16 都是完全平方数，而 3 和 11 不是。

解析

自己的

  1. 判断这个数n基础大小,小于4直接return n;
  2. 判断这个数本身是不是完全平方数,是直接return 1;
  3. 对每一个i都进行i*i~n之间的判断最优解,不只是单纯地判断离n最近的i进行求解
  4. 因为可能会出现19->16-1-1-1,但是19->9-9-1是更优解

但是这样的方式有点慢341ms,快的都在100ms以内,我是有点看不懂的,只有问看ai了

别人的

选择其中一个48ms的

我只能说简直就是优雅,思路听ai讲解和我的我觉得是一样的,但是写得又少时间又快;
理解这段代码dp[i] = Math.min(dp[i],dp[i-j*j]+1);
噢,我是将所有的j²<i的值都放在另一个数组里面,然后再从另一个数组里面进行遍历找最小值,
它这样直接min相当于节省了空间,又节省了另一个数组遍历的时间,我的天,简直就是聪明

总结

没什么好总结的,自己也是做出来了的,只不过做的过程当中有些细节还是没处理到位,
比如judge的值没变,一直都是false,导致每次的aaa都要全部变成100,
还有就是思路的问题,以及对于最优解的遍历,这些都是不足,慢慢学吧

300.最长递增子序列(medium)

数组,二分查找,动态规划

题目

给你一个整数数组 nums ，找到其中最长严格递增子序列的长度。
子序列 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。
例如，[3,6,2,7] 是数组 [0,3,1,6,2,2,7] 的子序列。

解析

经典的动态规划问题,我自己理解了动态规划之后做的两重循环解决,将时间复杂度变为了O(n²)

dep[i]=Math.max(dep[i],dep[j]+1);
这段代码我自己理解就是去找i前面的所有j,直到用max过滤出来作为dep[i]的值

总结

我看了有一个视频之后突然得到启发,是用表格形式可视化的展示了这个思路
(https://www.douyin.com/user/self?from_tab_name=main&modal_id=7524629126511152384&showTab=record)
就是突然灵光一现的感觉,之后觉得还是写下来没准会更好一点

322.零钱兑换(medium)

广度优先搜索,数组,动态规划

题目

给你一个整数数组 coins，表示不同面额的硬币；以及一个整数 amount，表示总金额。
计算并返回可以凑成总金额所需的 最少的硬币个数 。
如果没有任何一种硬币组合能组成总金额，返回 -1 。你可以认为每种硬币的数量是无限的。

解析

我选的思路还是算出这个coins里面从1~amount每种的最小值,然后比如当我计算到11的时候,
使用了一次5的硬币,那么剩下的6就已经在之前的计算当中了,就直接用dep[6]就行了

总结

这就是动态规划的魅力吗,感觉越来越有思路了,争取下一个题目by myself once accept,
前面或多或少都有ai提示思路或者不是一次过,修改了几遍

152.只出现一次的数字(easy)

位运算,数组

题目

给你一个非空整数数组 nums ，
除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。
你必须设计并实现线性时间复杂度的算法来解决此问题，且该算法只使用常量额外空间。

解析

本来是想用栈来解决的,但是这个不一定是顺序的,不像是括号那种,又想着先排序一下总行了吧,
但是那样时间复杂度好像就不是线性的了,最后确实想不到了,就问了AI了,居然是一种运算符
^异或运算🤟🤟🤟这个之前好像听说过,但是完全没在意

总结

异或运算就是二进制中的1,0相同就是0,不同就是1,这个非常实用这道题,
你将一个数转为二级制进行异或运算,比如x^y,然后后面又来了一个x,
那么之前相反了一次,这一次又相反,岂不就是y了吗,用这个运算符也是天才了都

设计一个算法来计算你所能获取的最大利润。
返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

其实这个很好算,我觉得和之前的动态规划一个样子,都是选逻辑中的某个特殊值,这个就是差的最大值
