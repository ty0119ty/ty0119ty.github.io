283.移动零(easy)

数组,双指针

题目

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。
请注意 ，必须在不复制数组的情况下原地对数组进行操作。

解析

将双指针看成是left,right这样两个,将数组分为两部分,
已处理的数组左侧and未处理数组的右侧

总结

假如前面都没有0,那么其实left一直都是right的,一旦出现0,那么left就必然和right出现差距,
只有而且left和right的差距也只会是中间0的个数,所以能完美解决这个问题

后续主包第二次做,思路是先找到有多少零,然后开始计数,不是零计数器+1,后面的前移到计数器位置

35.搜索插入位置(easy)

数组,二分查找

题目

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。
如果目标值不存在于数组中，返回它将会被按顺序插入的位置。
请必须使用时间复杂度为O(log n)的算法。

解析

主要看时间复杂度这里,不然直接一个遍历就出来了,说白了就是二分查找,
我觉得这里的就是左边界和右边界的的问题有点难理解,毕竟/2这个东西有点影响我思考数组下标,
其实就是想不清楚有些点

总结

就是算法基本功还不行吧,对数组的理解有待加强

49.字母异位词分组(mid)

哈希,map

题目

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。
字母异位词是通过重新排列不同单词或短语的字母而形成的单词或短语，并使用所有原字母一次。

解析

了解了一下哈希的东西,感觉一知半解吧,还有就是一些数据结构的操作不是很熟练

总结

感觉还是有点不熟,再加上自己本身对于算法/数据结构不是很精通,做这些题目都只能死脑筋做,
只有多多练习才行吧

128.最长连续序列(mid)

哈希,set

题目

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

解析

这里使用Set进行解决这个问题,问了ai说map也行,只不过set是单列map双列,这个问题set更优

总结

在这里我们可以知道hash查找为什么比遍历查找快是因为hash通过计算查找值,而遍历时通过比较查找;
12在有序正整数数组里面,要遍历12次才能找到,但是在hash的帮助下,只需要一次即可找到
别问为什么,因为主包也不知道为什么,这是在Hash里面封装好了的,我们只需要了解即可

11.盛最多水的容器(mid)

双指针

题目

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。
找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。
返回容器可以储存的最大水量。
说明：你不能倾斜容器。

解析

左指针和右指针向中心靠拢,因为如果h[0]<h[1],那么更大的面积只会有可能出现在更高的那一边(判断left和right哪边大了来)

总结

有点思路,但是是遍历,指针想到过,但是不知道为什么就能这样做,找了ai问了思路才知道的

15.三数之和(mid)

双指针

题目

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]]
满足 i != j、i != k 且 j != k ，
同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。
注意：答案中不可以包含重复的三元组。

解析

和11题类似,也是采用指针对冲进行优化效率,但是我们需要先进行对数组排序
