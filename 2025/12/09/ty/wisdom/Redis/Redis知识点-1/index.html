Redis知识点-1

发表于2025-12-09|更新于2025-12-09|Redis
|浏览量:

前言

作为后端开发中数据操作最主要的中间件之一,这也是基本开发需求所需要具备的能力

基础

什么是Redis说一下

Redis是一种基于键值对的NoSQL数据库(非关系型数据库);
它的主要特点就是把数据放在内存里面,提高读写速度,面对一些对性能要求很高的场景,
比如:缓存热点数据,频繁接口调用,都会用到Redis

Redis的数据类型

  1. String(字符串):可以用来存储token,验证码等等,常见于单个对象
  2. List(列表):可以用来缓存消息队列,任务列表等
  3. Hash(哈希):键值对集合,适合用来存储对象,如商品信息,用户信息等等
  4. Set(集合):查询效率很高,能达到O(1),主要用于去重,标签,共同好友等
  5. ZSet(有序集合):支持范围查询,常用于排行榜,优先级队列等

为什么Redis快

  1. 最简单直观的,数据存在内存里面,I/O操作相比于去磁盘里面,快了几个数量级
  2. Redis 采用了基于 I/O 多路复用技术的事件驱动模型来处理客户端请求和执行 Redis 命令
  3. Redis 对底层数据结构做了极致的优化,比如String的底层动态字符串支持动态扩容,预分配冗余空间,
    能够减少内存碎片和内存分配的开销

持久化

Redis的持久化方式

有两种方式:

  1. RDB:通过创建时间点快照来实现持久化
  2. AOF:通过记录每个写操作命令来实现持久化

这两种方式可以单独使用,也可以同时使用;
这样就可以保证 Redis 服务器在重启后不丢失数据,通过 RDB 和 AOF 文件来恢复内存中原有的数据

详细说一下RDB

RDB 持久化机制可以在指定的时间间隔内将 Redis 某一时刻的数据保存到磁盘上的 RDB 文件中,
当 Redis 重启时,可以通过加载这个 RDB 文件来恢复数据

详细说一下AOF

AOF 通过记录每个写操作命令,并将其追加到 AOF 文件来实现持久化,
Redis 服务器宕机后可以通过重新执行这些命令来恢复数据

Redis4.0的混合持久化了解吗

混合持久化结合了 RDB 和 AOF 两种方式的优点,解决了它们各自的不足

RDB恢复数据快,但是快照之间的时间间隔会导致数据丢失
AOF记录了每一次的操作,保证数据的持久性,但是一步一步恢复数据特别慢

原理就是先将RDB里面的数据加进去,再去找AOF中留存的数据;

缓存设计

缓存击穿,穿透,雪崩

缓存击穿

缓存击穿是指某个热点数据缓存过期时,大量请求就会穿透缓存直接访问数据库,导致数据库瞬间承受的压力巨大
解决方案:

  1. 互斥锁,当缓存失效的时候,第一个访问的线程先获取锁并负责重建缓存,其余线程等待/重试
  2. 永不过期策略,缓存项本身不设置过期时间,也就是永不过期,但在缓存值中维护一个逻辑过期时间;
    当缓存逻辑上过期时,返回旧值的同时,异步启动一个线程去更新缓存

缓存穿透

缓存穿透是指查询的数据在缓存中没有命中,因为数据压根不存在,所以请求会直接落到数据库上;
原因可能是自身的业务代码有问题,或者是恶意攻击造成的,比如爬虫
解决方案:

  1. 布隆过滤器,是一种空间效率很高的数据结构,可以用来判断一个元素是否在集合中
  2. 缓存空值,对于不存在的数据,我们将空值写入缓存,并设置一个合理的过期时间;
    这样下次相同的查询就能直接从缓存返回,而不再访问数据库

缓存雪崩

缓存雪崩是指在某一时间段,大量缓存同时失效或者缓存服务突然宕机了;
导致大量请求直接涌向数据库,导致数据库压力剧增,甚至引发系统崩溃的现象
成因及其解决方案:

  1. 大量缓存同时过期–>添加随机过期时间
  2. 缓存服务崩溃–>使用高可用的缓存集群
  3. 缓存服务正常但并发请求量超过了缓存服务的承载能力–>采用限流和降级措施

能说一下布隆过滤器吗

布隆过滤器是一种空间效率极高的概率性数据结构,用于快速判断一个元素是否在一个集合中;
它的特点是能够以极小的内存消耗,判断一个元素“一定不在集合中”或“可能在集合中”,
常用来解决 Redis 缓存穿透的问题

布隆过滤器可能会错误地认为某个元素在集合中,而元素实际上并不在集合中
但是,布隆过滤器认为某个元素不存在于集合中,那么它一定不存在

如何保证数据库和缓存数据一致性

读操作写入redis,增删改操作删除redis

读取时先查 Redis,未命中再查 MySQL,同时为缓存设置一个合理的过期时间;
更新时先更新 MySQL,再删除 Redis
这种方式简单有效,适用于读多写少的场景

Redis应用

Redis支持事务吗

Redis支持简单事务,可以将 multi、exec、discard 和 watch 命令打包,然后一次性的按顺序执行

  • 基本流程是用 multi 开启事务,然后执行一系列命令,最后用 exec 提交;
    这些命令会被放入队列,在 exec 时批量执行
  • discard 命令用于取消一个事务,它会清空事务队列并退出事务状态
  • watch 命令用于监视一个或者多个 key,如果这个 key 在事务执行之前 被其他命令改动,那么事务将会被打断
  • 但 Redis 的事务与 MySQL 的有很大不同,它并不支持回滚,也不支持隔离级别
